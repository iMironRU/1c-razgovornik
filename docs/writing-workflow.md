# Рабочий процесс

## Цикл написания параграфа

1. Написать параграф целиком (не дробить на куски — теряется связность)
2. Технический peer review: замечания по содержанию рассматриваются всегда
3. Правки точечно через `str_replace` — не переписывать параграф целиком
4. После значительных правок — сверка с соседними параграфами на согласованность
5. Статус `draft` → `review` → `ready`

## Что проверяет peer review

- Техническая корректность (примеры кода, поведение платформы)
- Корректность аналогий с другими языками
- Полнота: не пропущены ли важные ловушки

## Что НЕ меняет peer review

- Замечания по подаче, противоречащие стилю (например, «ужать»)
- Замечания, нарушающие принципы конституции
