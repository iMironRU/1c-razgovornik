# Руководство по стилю

## Структура параграфа

1. Связь с предыдущим (1–2 предложения)
2. Основной материал: описание в 1С → почему так
3. Таблица аналогий: 1С / Python / JavaScript / C#
4. Блок «Ловушки при переходе»
5. Якорная выноска `>` с главной мыслью

## Голос и тон

- Бытовой регистр + строгий термин в скобках: «Отобрать (отбор, фильтрация)»
- Метафоры с парной строгой формулировкой рядом
- Никаких абсолютов: не «без X нет Y», а «возникает, когда…»
- Двусторонние примеры: и «питонист смотрит на 1С», и «1С-разработчик смотрит на Python»

## Разметка

- Минимум разметки, максимум прозы
- Якорные мысли — выноска `>`
- Таблицы — только для сравнений кода
- ASCII-схемы для структурных диаграмм
- Mermaid — только если ASCII не справляется
- Никаких эмодзи в основном тексте

## Блоки кода

- Язык всегда указывается: `bsl`, `python`, `javascript`, `csharp`
- Комментарии в коде — объяснение результата: `// "510"`
- Ловушки оформляются блоком в конце параграфа, не разбросаны по тексту
