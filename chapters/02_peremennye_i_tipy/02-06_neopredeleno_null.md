## § 2.6 Неопределено и Null

В 1С два специальных значения для «ничего» — `Неопределено` и `Null`. Их легко перепутать, но они означают разные вещи.

**`Неопределено`** — значение переменной, которой ещё ничего не присвоено, или результат операции, не вернувшей значение. Это «внутреннее ничто» языка:

```bsl
Перем Х;
Сообщить(Х);         // Неопределено — переменная объявлена, но не инициализирована

Функция НайтиЧтоТо()
    // функция ничего не возвращает явно
КонецФункции

Результат = НайтиЧтоТо();
Сообщить(Результат);  // Неопределено
```

**`Null`** — значение, которое приходит из базы данных, когда поле не заполнено или запрос вернул пустое значение для внешнего соединения. Это «ничто» из мира данных:

```bsl
// Null появляется при работе с запросами и реквизитами
// В обычном коде Null явно не создаётся
```

```
Понятие          1С                Python     JavaScript       C#
───────────────  ────────────────  ─────────  ───────────────  ────────
Не инициализиров Неопределено      None       undefined        null / default
Пусто из БД      Null              None       null             null / DBNull
Проверка          Х = Неопределено  x is None  x === undefined  x == null
```

В JavaScript различие между `null` и `undefined` семантически близко к различию `Null` и `Неопределено` в 1С — но в JavaScript оба значения можно создать явно, а в 1С `Null` появляется только из запросов к данным.

> `Неопределено` — отсутствие значения в коде. `Null` — отсутствие значения в данных, пришедших из базы. Смешивать их не стоит: проверки пишутся через `= Неопределено` и `= Null` соответственно.
