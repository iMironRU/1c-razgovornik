## § 2.3 Строка

Строки в 1С заключаются в двойные кавычки. Многострочная строка записывается с символом `|` в начале каждой новой строки:

```bsl
// Однострочная
Приветствие = "Добрый день";

// Многострочная
Текст = "Уважаемый клиент,
        |Ваш заказ принят.
        |Спасибо за обращение.";
```

Символ `|` — не часть содержимого строки, а признак продолжения. В итоговой строке его нет.

**Конкатенация** выполняется оператором `+`. Здесь возвращается ловушка из § 1.3: если левый операнд строка, правый будет приведён к строке неявно. В обратную сторону правило не работает.

```bsl
Результат = "Итого: " + 1500;   // "Итого: 1500" — число приведено к строке
Результат = "5" + 10;           // "510"  — левый операнд строка, правый приведён к строке
Результат = 5 + "10";           // ОШИБКА — левый операнд число, строку к числу не приведёт
```

Сравнение строк в 1С выполняется без учёта регистра по умолчанию (точное поведение зависит от настроек информационной базы, но в большинстве конфигураций именно так):

```bsl
"Привет" = "привет"   // Истина
"ABC" = "abc"         // Истина
```

Это отличает 1С от Python и C#, где сравнение строк регистрозависимо. Для регистрозависимого сравнения в 1С нужна явная функция.

**Полезные функции работы со строками:**

```bsl
СтрДлина("привет")              // 6
СтрНайти("привет", "вет")       // 4 (позиция с 1, не с 0)
Лев("привет", 3)                // "при"
Прав("привет", 3)               // "вет"
Сред("привет", 3, 2)            // "ив"
СтрЗаменить("привет", "и", "е") // "привет" → "превет"
ВРег("привет")                  // "ПРИВЕТ"
НРег("ПРИВЕТ")                  // "привет"
СокрЛП("  текст  ")             // "текст" (trim с обеих сторон)
```

Разработчику из Python бросится в глаза нумерация с единицы, а не с нуля. Это системное решение 1С: индексация почти везде начинается с 1. Исключение — массив, где индексация с нуля. Это непоследовательность, о которой нужно просто знать.

> Строки сравниваются без учёта регистра. Индексация в строковых функциях начинается с 1. Многострочная строка оформляется символом `|` в начале продолжений.
