## § 2.1 Переменные: объявление и область видимости

В динамических языках переменная начинает существовать в момент первого присваивания. В Python достаточно написать `x = 5`. В JavaScript — `let x = 5` или просто `x = 5` в нестрогом режиме. В 1С есть нюанс, который зависит от того, где именно находится переменная.

**Внутри процедуры или функции** переменная создаётся первым присваиванием — без объявления:

```bsl
Процедура РассчитатьСтоимость()
    Цена = 1000;        // переменная создана
    Количество = 5;     // ещё одна
    Итого = Цена * Количество;
    Сообщить(Итого);
КонецПроцедуры
```

**На уровне модуля** — переменная, которая должна быть видна во всех процедурах модуля, объявляется явно через `Перем`:

```bsl
Перем СчётчикВызовов;   // объявлена на уровне модуля

Процедура ПервоеДействие()
    СчётчикВызовов = СчётчикВызовов + 1;
КонецПроцедуры

Процедура ВтороеДействие()
    Сообщить(СчётчикВызовов);  // та же переменная
КонецПроцедуры
```

Это похоже на атрибут класса в Python или переменную модуля в JavaScript — она живёт на уровне модуля и видна всем его процедурам.

```
Уровень             1С                  Python              JavaScript
──────────────────  ──────────────────  ──────────────────  ──────────────────
Внутри функции      просто присвоить    просто присвоить    let / const
Уровень модуля      Перем Имя;          просто присвоить    let вне функций
Экспорт наружу      Перем Имя Экспорт;  —                   export
```

**Константы** в 1С как языковой конструкции нет. Нет ключевого слова `const` или `final`. По соглашению константные значения оформляются либо как функции без параметров, возвращающие значение, либо просто именуются с заглавной буквы — но язык этого не контролирует.

Вместе с тем в 1С существуют **Константы как объект конфигурации** — это отдельная сущность уровня метаданных, не синтаксиса. Константа конфигурации хранится в базе данных, имеет тип и значение, задаётся в конфигураторе и доступна из любого модуля через `Константы.ИмяКонстанты.Получить()`. Именно так в реальных конфигурациях хранятся неизменяемые параметры системы — ставки, реквизиты организации, настройки. Разработчику из C# или JavaScript важно знать: константы *есть*, просто они живут не в синтаксисе языка, а в архитектуре конфигурации.

> Переменная внутри процедуры создаётся первым присваиванием. Переменная уровня модуля требует явного объявления через `Перем`. Констант как языковой конструкции нет.

**Ловушки при переходе:**

*Из C# в 1С.* Привычка объявлять все переменные в начале блока `int x; string name;` — в 1С это не нужно и не принято внутри процедур. `Перем` нужен только на уровне модуля.

*Из 1С в Python.* Отсутствие `Перем` кажется опасным — в Python тоже нет объявления, и это нормально. Но в Python переменная уровня модуля и локальная переменная функции разграничиваются ключевым словом `global` при изменении — в 1С этого нет, уровень определяется местом объявления `Перем`.
