## § 4.5 Цикл «Пока»

`Для…По` из § 4.3 ходит фиксированное число раз, `Для Каждого` из § 4.4 — по готовой коллекции. Когда заранее не известно, сколько будет итераций, или нужен нестандартный шаг, берут цикл по условию.

```bsl
Остаток = 100;
Пока Остаток > 0 Цикл
    Остаток = Остаток - 30;
    Сообщить(Остаток);
КонецЦикла;
```

`Пока условие Цикл … КонецЦикла`. Перед каждым проходом проверяется условие (опять `Булево`, как и в § 4.1); пока оно истинно, тело повторяется. Тело обязано рано или поздно сделать условие ложным — иначе цикл не закончится никогда.

`Пока` закрывает то, чего не умеет `Для…По`: шаг, отличный от +1, и движение в обратную сторону. Именно так в 1С удаляют элементы коллекции по условию (см. § 4.4) — обратным проходом по индексу:

```bsl
Сч = Товары.Количество() - 1;
Пока Сч >= 0 Цикл
    Если Товары[Сч] = "Карандаш" Тогда
        Товары.Удалить(Сч);     // безопасно: идём с конца, индексы впереди не сдвигаются
    КонецЕсли;
    Сч = Сч - 1;
КонецЦикла;
```

### Постусловия нет

Условие проверяется **до** тела. Поэтому если оно ложно с самого начала, тело не выполнится ни разу. Цикла с проверкой после тела — `do … while`, который гарантирует хотя бы один проход, — в 1С нет.

```bsl
Очередь = Новый Массив;        // пустая
Пока Очередь.Количество() > 0 Цикл
    Обработать(Очередь[0]);    // не выполнится ни разу — условие ложно сразу
    Очередь.Удалить(0);
КонецЦикла;
```

Если по смыслу один проход нужен обязательно, его выносят перед циклом или эмулируют флагом.

| | 1С | Python | JavaScript | C# |
|---|---|---|---|---|
| Цикл с предусловием | `Пока усл Цикл` | `while усл:` | `while (усл)` | `while (усл)` |
| Цикл с постусловием | нет | нет | `do { } while (усл)` | `do { } while (усл)` |
| Счётчик | вручную в теле | вручную | вручную | вручную |

> `Пока` проверяет условие до тела, поэтому при ложном условии тело не выполнится ни разу. Аналога `do…while` (проверка после тела, гарантированный первый проход) в 1С нет — его эмулируют отдельным шагом перед циклом.

**Ловушки при переходе:**

*Из Python в 1С.* Прямая аналогия `while`, расхождений мало. Помните лишь, что условие должно быть `Булево`: `Пока Очередь.Количество() Цикл` — ошибка, нужно `> 0`.

*Из JavaScript / C# в 1С.* `do…while` переносить некуда — переписывайте на `Пока` с предварительным проходом. Внутри `Пока` нет автоинкремента: счётчик наращивают руками, забытый `Сч = Сч + 1` даёт вечный цикл.

*Из 1С в другие языки.* Там, где в 1С пришлось разворачивать `Пока` ради нестандартного шага или обратного обхода, в Python это часто короче через `range(start, stop, step)` или `reversed()`.
