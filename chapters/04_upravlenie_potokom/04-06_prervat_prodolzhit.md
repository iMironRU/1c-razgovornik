## § 4.6 Прервать и Продолжить

Циклы из § 4.3–4.5 проходят до конца. Иногда нужно выйти раньше или пропустить один проход — для этого есть две команды.

```bsl
Для Каждого Товар Из Товары Цикл
    Если Товар.Остаток = 0 Тогда
        Продолжить;            // пропустить этот товар, перейти к следующему
    КонецЕсли;
    Если Товар.Код = ИскомыйКод Тогда
        Найден = Товар;
        Прервать;              // выйти из цикла совсем
    КонецЕсли;
КонецЦикла;
```

`Продолжить` бросает текущий проход и переходит к следующему — остаток тела до конца цикла пропускается. `Прервать` выходит из цикла целиком, управление идёт на строку после `КонецЦикла`. Работают в любом из трёх циклов одинаково.

### Метки и вложенность

Обе команды действуют только на **ближайший** цикл, внутри которого написаны. Меток у них нет — нельзя одной командой выйти сразу из двух вложенных циклов, как `break label;` в JavaScript или `break` к именованной метке.

```bsl
Найдено = Ложь;
Для Каждого Заказ Из Заказы Цикл
    Для Каждого Строка Из Заказ.Товары Цикл
        Если Строка.Товар = Искомый Тогда
            Найдено = Истина;
            Прервать;          // выходит только из внутреннего цикла
        КонецЕсли;
    КонецЦикла;
    Если Найдено Тогда
        Прервать;              // второй Прервать — для внешнего
    КонецЦикла;
КонецЦикла;
```

Выход из вложенных циклов собирают вручную: флаг плюс второй `Прервать`, как выше. Два других пути — вынести вложенные циклы в отдельную функцию и выйти из неё через `Возврат` (глава 5), либо, реже, через `Перейти` (§ 4.7).

| | 1С | Python | JavaScript | C# |
|---|---|---|---|---|
| Выйти из цикла | `Прервать` | `break` | `break` | `break` |
| Пропустить проход | `Продолжить` | `continue` | `continue` | `continue` |
| Выход из вложенных | флаг / `Возврат` | флаг / функция | `break label` | флаг / `goto` |

> `Прервать` и `Продолжить` действуют только на ближайший цикл, меток у них нет. Выход сразу из нескольких вложенных циклов собирают вручную — флагом со вторым `Прервать` или выносом в функцию с `Возврат`.

**Ловушки при переходе:**

*Из Python в 1С.* `break` → `Прервать`, `continue` → `Продолжить`, один к одному. Но у цикла в 1С нет ветки `else` (как `for…else` в Python), которая срабатывает при выходе без `break` — эту логику пишут флагом.

*Из JavaScript в 1С.* Помеченный `break outer;` не переносится — меток нет. Замена — флаг и второй `Прервать` или вынос в функцию.

*Из 1С в другие языки.* В JavaScript выход из вложенных циклов короче через метку, в Python и C# — тот же флаг или ранний `return`. Ручная конструкция из 1С переносится без потерь, но её часто можно упростить.
