## § 4.1 Условный оператор

Коллекции из главы 3 хранят данные. Чтобы программа что-то с ними делала по-разному в зависимости от ситуации, нужен выбор — условный оператор.

В 1С ветвление пишется словами, а не символами:

```bsl
Если Сумма > 1000 Тогда
    Скидка = 0.10;
ИначеЕсли Сумма > 500 Тогда
    Скидка = 0.05;
Иначе
    Скидка = 0;
КонецЕсли;
```

Каркас всегда один: `Если … Тогда`, затем тело, опциональные `ИначеЕсли … Тогда` и `Иначе`, и обязательный `КонецЕсли`. Ключевое слово `Тогда` отделяет условие от тела — без него синтаксическая ошибка. `ИначеЕсли` пишется слитно, в одно слово. Блок закрывается явно (`КонецЕсли`), а не отступом или скобкой — поэтому отступ в 1С косметический, а не синтаксический.

Логику внутри условия собирают тоже словами: `И`, `ИЛИ`, `НЕ`. Сравнение на равенство — один знак `=`, тот же, что и присваивание; платформа различает их по позиции в выражении.

```bsl
Если Клиент.Постоянный И Сумма > 1000 Тогда
    Скидка = 0.15;
КонецЕсли;
```

### Условие должно быть булевым

Главное отличие от Python и JavaScript: 1С не приводит значение к «истинности». Условие обязано иметь тип `Булево`. Число, строка или коллекция в роли условия — это ошибка времени выполнения, а не «ложь при нуле/пустоте».

```bsl
// Ошибка: Количество() возвращает Число, а не Булево
Если Товары.Количество() Тогда
    Обработать(Товары);
КонецЕсли;

// Правильно: явное сравнение даёт Булево
Если Товары.Количество() > 0 Тогда
    Обработать(Товары);
КонецЕсли;
```

Привычка `if items:` или `if (count)` здесь не работает — условие пишется до конца: `> 0`, `<> ""`, `<> Неопределено`.

### Множественного выбора нет

В 1С нет `switch`, `case` или `match`. Любой разбор по множеству значений — это цепочка `ИначеЕсли`:

```bsl
Если Статус = "Новый" Тогда
    Цвет = "Серый";
ИначеЕсли Статус = "ВРаботе" Тогда
    Цвет = "Жёлтый";
ИначеЕсли Статус = "Закрыт" Тогда
    Цвет = "Зелёный";
Иначе
    Цвет = "Чёрный";
КонецЕсли;
```

Когда веток много и все они — простое сопоставление «значение → значение», идиоматичная замена `switch` в 1С это не `Если`, а `Соответствие` (см. § 3.3): ключ ищется напрямую, без перебора условий.

```bsl
ЦветаПоСтатусу = Новый Соответствие;
ЦветаПоСтатусу["Новый"]    = "Серый";
ЦветаПоСтатусу["ВРаботе"]  = "Жёлтый";
ЦветаПоСтатусу["Закрыт"]   = "Зелёный";

Цвет = ЦветаПоСтатусу[Статус];
```

| Конструкция | 1С | Python | JavaScript | C# |
|---|---|---|---|---|
| Условие | `Если … Тогда` | `if …:` | `if (…)` | `if (…)` |
| Иначе-если | `ИначеЕсли … Тогда` | `elif …:` | `else if (…)` | `else if (…)` |
| Иначе | `Иначе` | `else:` | `else` | `else` |
| Конец блока | `КонецЕсли;` | отступ | `}` | `}` |
| И / ИЛИ / НЕ | `И` `ИЛИ` `НЕ` | `and` `or` `not` | `&&` `\|\|` `!` | `&&` `\|\|` `!` |
| Равенство | `=` | `==` | `===` | `==` |
| Множественный выбор | нет | `match` | `switch` | `switch` |

> Условие в `Если` обязано быть типа `Булево` — 1С не считает ноль, пустую строку или пустую коллекцию за «ложь». А `switch` в языке отсутствует: его роль играют либо цепочка `ИначеЕсли`, либо `Соответствие`.

**Ловушки при переходе:**

*Из Python в 1С.* `if товары:` не транслируется напрямую — пустой массив не «ложь». Пишите `Если Товары.Количество() > 0 Тогда`. Оператор `match` заменяется цепочкой `ИначеЕсли` или `Соответствием`.

*Из JavaScript в 1С.* Сравнение — один знак `=`, а не `===`; строгого и нестрогого равенства в 1С нет. Логические операторы — слова `И`, `ИЛИ`, `НЕ`, а не `&&`, `||`, `!`. Привычка к `switch` упирается в его отсутствие.

*Из 1С в другие языки.* `Иначе` — это `else`, а не `else if`; для второй проверки нужен отдельный `elif` / `else if`. И наоборот: то, что в 1С приходилось расписывать цепочкой `ИначеЕсли`, в Python и C# часто короче через `match` / `switch`.
