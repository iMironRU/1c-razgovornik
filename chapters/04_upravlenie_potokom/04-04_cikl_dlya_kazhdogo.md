## § 4.4 Цикл по коллекции

Счётчик из § 4.3 нужен, когда важен номер. Чаще номер не нужен — нужен сам элемент. Для этого есть обход коллекции.

```bsl
Для Каждого Товар Из Товары Цикл
    Сообщить(Товар);
КонецЦикла;
```

Переменная `Товар` на каждом проходе получает очередной элемент коллекции `Товары`. Индекса нет вообще — это его главное отличие от `Для…По` и причина, по которой такой обход предпочтительнее, когда позиция элемента не важна.

Работает с любой коллекцией из главы 3. Но то, чем окажется элемент, зависит от типа коллекции:

```bsl
// Массив, Список значений — элемент это само значение
Для Каждого Товар Из Товары Цикл
    Сообщить(Товар);
КонецЦикла;

// Структура, Соответствие — элемент это пара КлючИЗначение
Для Каждого КлючЗначение Из Остатки Цикл
    Сообщить(КлючЗначение.Ключ + ": " + КлючЗначение.Значение);
КонецЦикла;

// Таблица значений — элемент это строка, к колонкам по имени
Для Каждого Строка Из ТаблицаПродаж Цикл
    Сообщить(Строка.Товар + " — " + Строка.Сумма);
КонецЦикла;
```

### Менять коллекцию во время обхода нельзя

Пока идёт `Для Каждого`, добавлять или удалять элементы перебираемой коллекции запрещено — платформа прервёт работу ошибкой. Это страховка от обхода, который сам у себя выбивает почву из-под ног.

```bsl
// Ошибка: удаление во время обхода
Для Каждого Товар Из Товары Цикл
    Если Товар = "Карандаш" Тогда
        Товары.Удалить(Товары.Найти(Товар));   // прервётся ошибкой
    КонецЕсли;
КонецЦикла;
```

Если элементы нужно удалять по условию, обходят по индексу в обратную сторону через `Пока` (§ 4.5) — так удаление не сдвигает ещё не пройденные элементы, — либо собирают «на удаление» в отдельный массив, а чистят вторым проходом.

| | 1С | Python | JavaScript | C# |
|---|---|---|---|---|
| Запись | `Для Каждого э Из к Цикл` | `for э in к:` | `for (const э of к)` | `foreach (var э in к)` |
| Доступ к индексу | нет | `enumerate(к)` | `.entries()` | — |
| Пара ключ-значение | `.Ключ` / `.Значение` | `.items()` | `Object.entries` | `KeyValuePair` |
| Правка при обходе | запрещена (ошибка) | запрещена | допустима, но опасна | запрещена (исключение) |

> `Для Каждого` даёт элемент, но не его номер. Менять перебираемую коллекцию во время обхода нельзя — удаление по условию делают обратным проходом через `Пока` или вторым проходом по отдельному списку.

**Ловушки при переходе:**

*Из Python в 1С.* Индекса нет — `enumerate` не транслируется; если номер нужен, берите `Для…По` (§ 4.3). У `Структуры` и `Соответствия` элемент — это объект с `.Ключ` и `.Значение`, а не кортеж `(k, v)` для распаковки.

*Из JavaScript в 1С.* `for…of` правку коллекции на ходу терпит, `Для Каждого` — нет, сразу ошибка. Это не «опасно, но можно», а жёсткий запрет.

*Из 1С в другие языки.* Запрет на правку при обходе есть и в C# (`InvalidOperationException`), а в JS — нет. Не переносите в JS уверенность, что среда сама вас остановит.
