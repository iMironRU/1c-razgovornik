## § 4.3 Цикл со счётчиком

Выбор из § 4.1 и § 4.2 решал, что сделать один раз. Чтобы повторить действие заданное число раз, нужен цикл со счётчиком.

В 1С он выглядит так:

```bsl
Для Сч = 1 По 3 Цикл
    Сообщить("Сотрудник №" + Сч);
КонецЦикла;
```

Счётчик `Сч` пробегает от начального значения до конечного включительно. `Для Сч = 1 По 3` — это три прохода: 1, 2, 3. Блок закрывается `КонецЦикла`.

### Граница включается

Здесь прячется самая частая ловушка перехода. В 1С верхняя граница **входит** в диапазон: `По 3` означает «и тройку тоже». В Python `range(3)` даёт `0, 1, 2` — тройки нет; граница исключается. То есть в `По` пишут **последний индекс**, а не количество шагов.

Поэтому перебор массива (индексация с нуля, см. § 3.1) выглядит так:

```bsl
Для Сч = 0 По Товары.Количество() - 1 Цикл
    Сообщить(Товары[Сч]);
КонецЦикла;
```

`- 1` обязателен: будь там `По Товары.Количество()`, цикл зашёл бы на индекс, которого нет, и упал с ошибкой выхода за границу. Привычка из Python писать `range(len(товары))` без поправки здесь даёт ошибку, а не молчаливый пропуск.

### Шаг только +1, граница считается один раз

Параметра шага у `Для…По` нет — счётчик растёт строго на единицу. Идти в обратную сторону или через два этот цикл не умеет; для такого берут `Пока` (§ 4.5).

Верхняя граница вычисляется один раз, в начале цикла. Если внутри тела изменить то, из чего она получена, число итераций не поменяется:

```bsl
Предел = 3;
Для Сч = 1 По Предел Цикл
    Предел = 100;          // на число итераций не влияет — границу уже посчитали
    Сообщить(Сч);          // три прохода: 1, 2, 3
КонецЦикла;
```

| | 1С | Python | JavaScript | C# |
|---|---|---|---|---|
| Запись | `Для Сч = 0 По N Цикл` | `for сч in range(N+1):` | `for (let i=0;i<=N;i++)` | `for (int i=0;i<=N;i++)` |
| Граница | включается | исключается | задаётся вручную | задаётся вручную |
| Шаг | всегда +1 | любой (`range` step) | любой | любой |
| Пересчёт границы | один раз | — | каждую итерацию | каждую итерацию |

> Верхняя граница в `Для…По` включается: `По 9` — это 10 итераций. Перебирая коллекцию, пишите `По Количество() - 1`. Шаг всегда +1, а сама граница вычисляется однократно в начале.

**Ловушки при переходе:**

*Из Python в 1С.* `range(N)` исключает `N`, `По N` — включает. Прямой перенос `range(len(а))` в `По а.Количество()` выходит за границу массива. Шага нет: `range(0, 100, 10)` переписывается через `Пока` с ручным `+= 10`.

*Из JavaScript / C# в 1С.* Условие и инкремент не пишутся — только начало и конец. Декремент (`i--`) невозможен; обратный обход — через `Пока`.

*Из 1С в другие языки.* `По N` это `<= N`, а не `< N`: при переносе в `for (i=0; i<?; i++)` нужно `<= N` или `< N+1`, иначе потеряется последняя итерация.
