## § 4.2 Тернарный оператор

Когда `Если` из § 4.1 нужен лишь для того, чтобы выбрать одно из двух значений, разворачивать пять строк необязательно — есть короткая форма.

В 1С это `?()`:

```bsl
Скидка = ?(Сумма > 1000, 0.10, 0);
```

Читается как «если `Сумма > 1000`, то `0.10`, иначе `0`». Первый аргумент — условие (снова обязан быть `Булево`, как в § 4.1), второй — значение при истине, третий — при лжи.

Внешне это тернарный оператор из C-подобных языков, но устроен он иначе. `?()` — не оператор, а встроенная **функция**. И отсюда следует главная особенность: оба значения вычисляются всегда, ещё до того, как проверится условие. Аргументы функции в 1С считаются до вызова — лениво, «по одной ветке», как `c ? a : b` в JavaScript, `?()` не работает.

```bsl
// Опасно: деление выполнится ДАЖЕ когда Количество = 0
ЦенаЗаШтуку = ?(Количество <> 0, Сумма / Количество, 0);   // ошибка «Деление на 0»
```

Здесь `Сумма / Количество` вычисляется независимо от условия. Если `Количество` равно нулю, деление произойдёт и вызовет ошибку — третья ветка `0` не спасёт. Для таких случаев нужен обычный `Если`:

```bsl
Если Количество <> 0 Тогда
    ЦенаЗаШтуку = Сумма / Количество;
Иначе
    ЦенаЗаШтуку = 0;
КонецЕсли;
```

Правило простое: `?()` хорош, когда обе ветки — это готовые безопасные значения. Как только в ветке появляется деление, вызов метода с побочным эффектом или обращение к тому, чего может не быть, — берите `Если`.

| | 1С | Python | JavaScript | C# |
|---|---|---|---|---|
| Запись | `?(усл, а, б)` | `а if усл else б` | `усл ? а : б` | `усл ? а : б` |
| Что это | функция | оператор | оператор | оператор |
| Вычисление веток | обе всегда | только нужную | только нужную | только нужную |

> `?()` в 1С — функция, а не ленивый оператор: обе ветки вычисляются всегда. Если в ветке возможна ошибка или побочный эффект, тернарник не годится — нужен `Если`.

**Ловушки при переходе:**

*Из Python в 1С.* `а if усл else б` вычисляет только выбранную ветку — `?()` вычисляет обе. Перенос выражения с делением или вызовом, защищённого условием, ломается. Порядок аргументов тоже другой: в 1С условие идёт **первым**.

*Из JavaScript в 1С.* `усл ? а : б` короткозамкнут, `?(усл, а, б)` — нет. Идиома `х ?? умолчание` и `х?.свойство` в 1С отсутствуют; проверку на `Неопределено` пишут явным `?()` или `Если`.

*Из 1С в другие языки.* Двусторонне безопасно: раз `?()` всё равно считал обе ветки, при переносе в `c ? a : b` поведение не ухудшится. Но можно и упростить — то, что в 1С пришлось выносить в `Если` из-за жадности `?()`, в Python и JS снова сворачивается в одну строку.
