## § 3.7 Фиксированные коллекции

Для массива, структуры и соответствия существуют неизменяемые версии: **ФиксированныйМассив**, **ФиксированнаяСтруктура**, **ФиксированноеСоответствие**. Создаются из обычных коллекций — после создания изменить их нельзя.

```bsl
// Обычный массив → фиксированный
Обычный = Новый Массив;
Обычный.Добавить("Ручка");
Обычный.Добавить("Карандаш");

Фиксированный = Новый ФиксированныйМассив(Обычный);
Фиксированный[0];           // "Ручка" — чтение работает
Фиксированный.Добавить(…);  // ОШИБКА — изменение запрещено
```

| Неизменяемая коллекция | 1С | Python | JavaScript | C# |
|---|---|---|---|---|
| Массив | `ФиксированныйМассив` | `tuple` | `Object.freeze([])` | `ReadOnlyCollection<T>` |
| Структура | `ФиксированнаяСтруктура` | `namedtuple` | `Object.freeze({})` | — |
| Соответствие | `ФиксированноеСоответствие` | `MappingProxyType` | `Object.freeze(Map)` | `ReadOnlyDictionary<>` |

Фиксированные коллекции используются там, где нужно передать набор данных и гарантировать, что получатель его не изменит: настройки, справочные данные, параметры конфигурации.

> Фиксированные коллекции создаются из обычных и после создания не изменяются. Это аналог `tuple` и `readonly` в других языках — контракт неизменности, выраженный в типе.
