## § 3.4 Список значений

Список значений — коллекция, где каждый элемент содержит не одно, а четыре поля: **Значение**, **Представление**, **Пометка** и **Картинка**. Это не просто список — это список с метаданными для каждого элемента.

```bsl
СписокТоваров = Новый СписокЗначений;

// Добавить элемент
СписокТоваров.Добавить("Ручка", "Шариковая ручка синяя", Ложь);
//                      Значение  Представление             Пометка

// Добавить в нужную позицию
СписокТоваров.Вставить(0, "Карандаш", "Простой карандаш", Истина);

// Доступ к элементу
Элемент = СписокТоваров[0];
Сообщить(Элемент.Значение);          // "Карандаш"
Сообщить(Элемент.Представление);     // "Простой карандаш"
Сообщить(Элемент.Пометка);           // Истина

// Перебор
Для Каждого Элемент Из СписокТоваров Цикл
    Если Элемент.Пометка Тогда
        Сообщить(Элемент.Представление);
    КонецЕсли;
КонецЦикла;

// Полезные методы
СписокТоваров.СортироватьПоЗначению();
СписокТоваров.СортироватьПоПредставлению();
НайденныйЭлемент = СписокТоваров.НайтиПоЗначению("Ручка");
```

Прямого аналога в других языках нет. По структуре это ближе всего к списку именованных кортежей в Python:

```python
# Python — ближайший аналог через namedtuple
from collections import namedtuple
Элемент = namedtuple('Элемент', ['значение', 'представление', 'пометка'])
список = [
    Элемент("Ручка", "Шариковая ручка синяя", False),
    Элемент("Карандаш", "Простой карандаш", True),
]
```

Список значений часто используется там, где нужно передать набор вариантов: для заполнения выпадающего списка на форме, для передачи набора выбранных элементов, для промежуточного хранения пар «значение + человекочитаемое название» в алгоритме.

> Список значений — это список с четырьмя полями на элемент. Он используется не только в UI: поле «Пометка» удобно для алгоритмов выбора и фильтрации, поле «Представление» — для хранения человекочитаемого названия рядом с техническим значением.
