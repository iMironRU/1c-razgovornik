## § 3.3 Соответствие

Соответствие — коллекция пар «ключ : значение», где **ключ может быть любого типа**: строка, число, дата, ссылка на объект базы данных. Это главное отличие от Структуры.

```bsl
// Создание
Цены = Новый Соответствие;
Цены.Вставить("Ручка", 45);           // ключ — строка
Цены.Вставить(1001, 120);              // ключ — число
Цены.Вставить(СсылкаНаТовар, 350);    // ключ — ссылка на объект БД

// Получение
Цена = Цены["Ручка"];                  // 45
Цена = Цены[СсылкаНаТовар];           // 350

// Если ключа нет — возвращает Неопределено, не ошибку
НеизвестныйТовар = Цены["Что-то"];    // Неопределено

// Проверка
Если Цены["Ручка"] <> Неопределено Тогда ...

// Перебор
Для Каждого Элемент Из Цены Цикл
    Сообщить(Элемент.Ключ + " -> " + Элемент.Значение);
КонецЦикла;
```

| Операция | 1С Соответствие | Python dict | JavaScript Map | C# Dictionary |
|---|---|---|---|---|
| Создать | `Новый Соответствие` | `{}` | `new Map()` | `new Dictionary<>()` |
| Добавить | `.Вставить(к, в)` | `d[k] = v` | `.set(k, v)` | `d[k] = v` |
| Получить | `[ключ]` | `d[k]` | `.get(k)` | `d[k]` |
| Несуществующий | `Неопределено` | `KeyError` | `undefined` | `KeyNotFoundException` |
| Тип ключа | **любой** | любой | любой | ограничен параметром `TKey` |

Ближайший аналог по поведению — `Map` в JavaScript: оба позволяют использовать объекты как ключи и возвращают «пустое» значение при отсутствии ключа, не выбрасывая исключение.

> Соответствие отличается от Структуры одним: тип ключа не ограничен строкой-идентификатором. Когда ключ — ссылка на объект базы данных, число или что-то нестандартное — нужно Соответствие.

**Ловушки при переходе:**

*Из Python в 1С.* В Python обращение к несуществующему ключу словаря даёт `KeyError`. В 1С Соответствие возвращает `Неопределено` — молча. Это может скрыть ошибку в логике.

*Из C# в 1С.* Аналогично: нет исключения при отсутствии ключа. Проверка — через `<> Неопределено`.
