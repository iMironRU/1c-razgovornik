## § 3.5 Таблица значений

Таблица значений — самый мощный и самый специфичный для 1С тип коллекции. По структуре это двумерная таблица: именованные столбцы произвольного типа и произвольное количество строк. По возможностям — нечто среднее между `DataFrame` в pandas и `DataTable` в C#.

```bsl
// Создание и определение структуры
ТЗ = Новый ТаблицаЗначений;
ТЗ.Колонки.Добавить("Наименование");
ТЗ.Колонки.Добавить("Цена");
ТЗ.Колонки.Добавить("Количество");

// Добавление строк
НоваяСтрока = ТЗ.Добавить();
НоваяСтрока.Наименование = "Ручка";
НоваяСтрока.Цена         = 45;
НоваяСтрока.Количество   = 100;

НоваяСтрока = ТЗ.Добавить();
НоваяСтрока.Наименование = "Карандаш";
НоваяСтрока.Цена         = 20;
НоваяСтрока.Количество   = 200;

// Перебор строк
Для Каждого Строка Из ТЗ Цикл
    Итого = Строка.Цена * Строка.Количество;
    Сообщить(Строка.Наименование + ": " + Итого);
КонецЦикла;

// Поиск строки
НайденнаяСтрока = ТЗ.Найти("Ручка", "Наименование");
Если НайденнаяСтрока <> Неопределено Тогда
    НайденнаяСтрока.Цена = 50;  // изменение на месте
КонецЕсли;

// Отбор строк по условию
ОтборПараметры = Новый Структура("Наименование", "Ручка");
НайденныеСтроки = ТЗ.НайтиСтроки(ОтборПараметры);

// Сортировка
ТЗ.Сортировать("Цена Убыв");

// Итоги
ТЗ.Итог("Цена")         // сумма по столбцу
```

Таблица значений — серверный тип. Как передать её на клиент через границу контекстов — разобрано в § 1.5: преобразовать в массив структур.

```
Возможность              ТЗ (1С)        DataFrame (Python)    DataTable (C#)
──────────────────────   ───────────    ──────────────────    ──────────────
Именованные столбцы      Да             Да                    Да
Поиск по значению        Да             Да (.query)           Да (.Select)
Сортировка               Да             Да (.sort_values)     Да
Итоги по столбцу         Да             Да (.sum)             Да
Индексация строк         Нет            Да                    Нет
Векторные операции       Нет            Да                    Нет
Сериализуемость          Нет            Нет (напрямую)        Нет
```

> Таблица значений — серверный тип без прямых аналогов. Она умеет искать, сортировать и считать итоги, но не умеет векторных операций как pandas и не передаётся на клиент напрямую.

**Ловушки при переходе:**

*Из pandas в 1С.* Нет `df[df['Цена'] > 100]`. Отбор только через `.НайтиСтроки()` по точному совпадению или через явный цикл.

*Из 1С в pandas.* `тз.Итог("Цена")` — одна строка. В pandas `df['Цена'].sum()` — тоже одна строка, но синтаксис другой.

*Из любого языка в 1С.* Попытка передать ТЗ на клиент — ошибка выполнения. Нужно конвертировать заранее.
