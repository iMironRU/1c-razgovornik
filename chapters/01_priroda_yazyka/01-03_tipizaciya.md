## § 1.3 Типизация: мягкая, но не безграничная

В 1С нет объявления типа переменной. Переменная получает тип в момент присваивания значения и может изменить его при следующем присваивании. В этом 1С похожа на Python и JavaScript.

```bsl
// 1С
Перем Х;
Х = 42;        // Х — число
Х = "привет";  // Х — строка, без ошибки
Х = Истина;    // Х — булево, без ошибки
```

```python
# Python — то же самое, без объявления
x = 42
x = "привет"
x = True
```

```javascript
// JavaScript — аналогично
let x = 42;
x = "привет";
x = true;
```

```csharp
// C# — так нельзя: тип зафиксирован при объявлении
int x = 42;
x = "привет"; // ошибка компиляции
```

Однако 1С жёстче JavaScript в вопросах неявного приведения типов. JavaScript агрессивно приводит типы в выражениях — это источник бесконечных сюрпризов. 1С ведёт себя сдержаннее.

**Оператор `+` с участием строки** — главная ловушка при переходе:

```
Выражение        1С          Python      JavaScript
───────────────  ──────────  ──────────  ──────────
"" + 1           "1"         TypeError   "1"
"5" + 2          "52"        TypeError   "52"
5 + "10"         ОШИБКА      TypeError   15
5 + 2            7           7           7
```

В 1С действует асимметричное правило: тип *левого* операнда определяет поведение. Если левый операнд строка — правый будет приведён к строке. Если левый операнд число — строка справа к числу не приводится, возникает ошибка. Это не «магия» в стиле JavaScript, где строка побеждает в любой позиции, и не жёсткий запрет в стиле Python — это правило левого операнда.

**Явное приведение** выполняется функциями платформы:

| Цель | 1С | Python | JavaScript | C# |
|---|---|---|---|---|
| В строку | `Строка(Х)` | `str(x)` | `String(x)` | `x.ToString()` |
| В число | `Число(Х)` | `int(x)` / `float(x)` | `Number(x)` | `Convert.ToInt32(x)` |
| В булево | `Булево(Х)` | `bool(x)` | `Boolean(x)` | `Convert.ToBoolean(x)` |
| В дату | `Дата(Х)` | `datetime.fromisoformat(x)` | `new Date(x)` | `DateTime.Parse(x)` |

Отдельного внимания заслуживает тип **Дата**. В Python дата — это класс из библиотеки `datetime`. В JavaScript — объект `Date`. В C# — структура `DateTime` из стандартной библиотеки. В 1С дата является **встроенным примитивным типом**, таким же базовым, как число или строка. Литерал даты записывается апострофами: `'20240115'` или `'20240115120000'` (с временем). Операции с датами доступны без импорта чего-либо — платформа знает о датах с рождения.

Для разработчика на 1С, смотрящего на Python, отсутствие встроенной даты выглядит странно: как базовый тип может требовать импорта? Ответ исторический — Python проектировался как язык общего назначения, а не инструмент учёта. Дата в нём не базовая потребность, а одна из многих.

> В 1С нет объявления типа, но есть конкретное правило неявного приведения: строковый левый операнд `+` приводит правый к строке — в обратную сторону приведение не работает. Дата — встроенный примитивный тип.
